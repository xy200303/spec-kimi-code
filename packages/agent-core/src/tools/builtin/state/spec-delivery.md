Create the project-local delivery record for the active spec coding run. The record uses the immutable specification and design snapshot captured at approval time, including the approved goal, constraints, and acceptance criteria. It also records whether the run was approved automatically or by the user, when approval occurred, and any selected plan option. The approved plan's automatically selected development strategy and its recommendation are included in the record.

Call this after implementation has produced task, change, and tool-result traces. The record maps each changed file to its task, reason, risk, strategy category, and observed tool calls. Its Activity section records every traced command and delegated Agent call with outcome, task, command or delegation, changed files, and background status. Every quality-gate evidence item must reference its successful foreground `Bash` `toolCallId` from a completed spec task; a background command or command from another task is not proof that verification has finished. The completed task list must also cover every required category from the selected development strategy. Provide any decisions, risks, open questions, or rollback notes that are not already present in the specification and design. Set `complete` to `true` only when every spec task is done, every strategy category is covered, and every evidence category required by the selected quality gate has a verified tool call from a completed task. Completing the record writes a finalization time to both artifacts and locks them; start a new spec run for later changes. The tool writes matching human-readable and machine-readable records to `delivery.md` and `delivery.json` in the run directory.
