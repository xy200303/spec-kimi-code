Read the immutable snapshot and approval audit of the approved spec coding run. Use this before or during implementation to keep work aligned with the approved goal, constraints, acceptance criteria, selected development strategy, quality gate, planned tasks, current task status, planned risks, verification approach, approval source, timestamp, selected plan option, and delivery finalization time. The tool also compares the current specification and design documents with the approved snapshot and reports any drift; delivery records always use the approved snapshot.

This tool accepts no parameters and has no side effects. It is available only after `ExitPlanMode` has captured an approved spec run. If the plan is still being drafted or was not approved, revise the project-local specification and design documents instead.
