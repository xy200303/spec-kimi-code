Read the immutable snapshot and approval audit of the approved spec coding run. Use this before or during implementation to keep work aligned with the approved goal, constraints, acceptance criteria, selected development strategy, quality gate, planned tasks, verification approach, approval source, timestamp, and selected plan option.

This tool accepts no parameters and has no side effects. It is available only after `ExitPlanMode` has captured an approved spec run. If the plan is still being drafted or was not approved, revise the project-local specification and design documents instead.
